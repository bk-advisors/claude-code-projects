Core Insight: In a world of resource scarcity, a decision about which program elements to control or change must precede any decision on how to control or change them. Understanding causality is crucial to making effective interventions.

Situational Assessment

What is happening? What are the causes?

The Manager’s Two Questions

Every manager faces two fundamental questions:

Question 1: What is happening?

Situational Assessment — understanding the current state by gathering evidence and reasoning about causes.

“Why is neonatal mortality still high in District X despite our investments?”

Question 2: What should I do?

Managerial Intervention — choosing which levers to pull to change outcomes.

“Should we invest in more CPAP machines or in training more midwives?”

Situational Assessment in Practice

Situational Assessment: Using evidence to reason about the current state of a system — determining which variables are in which states, and why.

  • A police inspector assessing cause of death: What evidence points to which explanation?
  • A technical advisor diagnosing an organization’s declining health outcomes: Which factors are driving the problem?
  • A program director in a large MNH initiative asking: Why is a country’s NMR not declining?

Causal models help us incorporate multiple sources of evidence and reason systematically about what they imply.

The Maxwell Example (from the Book)

Ryall & Bramson open with a compelling example:

Robert Maxwell — media tycoon found dead off his yacht in 1991.

  • McKinsey’s issue tree approach: break down “How did he die?” into MECE categories
  • But this misses the causal question: What caused each possibility?
  • Causal thinking leads us both up the chain (what caused X?) and down (what evidence would X produce?)

Key Insight: Framing analysis in causal terms points us toward useful sources of information and allows us to incorporate evidence in useful ways.

Managerial Intervention

What should I do? Which levers should I pull?

Managers Manage Scarce Resources

Managerial Intervention: A decision about which objects to control or change — the choice of what to act on, before deciding how to act.

  • Resources are always limited — money, time, personnel, equipment
  • Every intervention involves trade-offs: investing in CPAP means not investing in workforce
  • Constraints imply choices, and choices require understanding of what causes what

A large-scale MNH initiative may deploy pooled investments across multiple countries in sub-Saharan Africa. Every dollar spent on one intervention is a dollar not spent on another. Which interventions will cause the greatest reduction in maternal and newborn mortality?

The Feather Touch Cautionary Tale

The book’s centerpiece example (adapted to our MNH context):

The Setup: A consulting organization (TruSmartz) analyzes data on an MNH program across multiple countries. They find:

  • Countries with more demand generation for health services show lower service utilization
  • Countries that run subsidy programs also show lower utilization
  • The regression model predicts: stop demand generation to increase utilization!

What went wrong? The regression captured correlations in the data, not causal effects. Demand generation and subsidies were substitutes in program managers’ decision-making — creating a spurious negative correlation.

The Causal Truth

The true causal model reveals a different story:

Demand Generation
Decision
→ [−]
Subsidy
Decision
→ [+]
Care-Seeking
Behavior
→ [+]
Service
Utilization
  • Program managers only run subsidies when they are not doing demand generation (substitutes!)
  • Subsidies drive care-seeking behavior (service uptake) — but they’re underused
  • Correct intervention: Run subsidies alongside demand generation, not as a replacement
  • The single-stage regression gets the sign wrong; the causal model gets it right

Regression vs. Causal Model: The Numbers

Approach Stop Subsidies Stop Demand Generation Run Both
Single-stage regression +73 impact units +282 impact units —
Multi-stage (causal) regression −783 impact units −57 impact units +544 impact units
True causal effect −832 impact units +38 impact units +816 impact units

The regression without causal structure recommends the worst option.

A causal model correctly identifies the best intervention.

Correlation ≠ Causation

The three traps of causal reasoning

Trap 1: Confounding

Confounding occurs when a third variable influences both the supposed cause and effect, creating a spurious association.

MNH Example

Observation: Countries with more health workers deployed have higher neonatal mortality rates.

Naive conclusion: Health workers cause deaths?!

Reality: More health workers are deployed to sicker areas (high-burden districts). Disease burden is the confounder — it drives both worker deployment and mortality.

Health Worker
Deployment
←
Disease
Burden
→
Neonatal
Mortality

Trap 2: Reverse Causation

Reverse Causation occurs when the supposed effect actually causes the supposed cause.

MNH Example

Observation: Countries receiving more donor funding have higher mortality rates.

Naive conclusion: Donor funding increases mortality?

Reality: High mortality attracts donor funding. The causal arrow runs in the opposite direction.

Assumed model:

Donor Funding → Higher Mortality

Actual model:

Higher Mortality → More Donor Funding

Trap 3: Selection Bias

Selection Bias occurs when the data sample is not representative of the population, because selection into the sample is related to both cause and effect.

MNH Example

Observation: Among hospitals that report data to DHIS2, CPAP usage appears unrelated to neonatal outcomes.

Problem: Only well-resourced hospitals report to DHIS2. These hospitals already have better outcomes — the data misses the facilities where CPAP would make the biggest difference.

Selection mechanism: Hospital capacity → {DHIS2 Reporting, Outcomes}

Three Traps Summary

Trap What Happens MNH Example
Confounding Third variable drives both X and Y Disease burden drives both worker deployment and mortality
Reverse Causation Y causes X, not X causes Y Mortality attracts funding, not vice versa
Selection Bias Non-random sample distorts relationship Only high-capacity hospitals report data

All three traps create statistical associations that look like causal effects but aren’t.

The antidote is explicit causal modeling.

The Causal Model Framework

Nodes, arrows, and directed graphs

Elements of a Causal Model

A causal model is a directed graph consisting of:

  • Nodes — variables in the system (represented by shapes)
  • Directed links (arrows) — causal relationships from one variable to another
  • Signs — [+] or [−] indicating the direction of effect

The key rule: An arrow from A to B means that changing A can change B, but changing B does not change A.

Directed Acyclic Graph (DAG): A causal diagram with no cycles — you cannot follow arrows from any node back to itself. This is the mathematical foundation of causal modeling.

A Simple MNH Causal Model

Investment
($M)
→ [+]
Intervention
Coverage (%)
→ [−]
Neonatal
Mortality Rate
  • Investment → Coverage [+]: More funding increases the percentage of facilities with key interventions
  • Coverage → NMR [−]: Higher intervention coverage reduces neonatal mortality
  • This is a serial chain — investment has an indirect effect on mortality through coverage

But this model is too simple! What about confounders, competing causes, feedback?

That’s what the rest of the book is about.

MNH Application: The Investment Question

Causal thinking for MNH program design

The MNH Investment Challenge

Consider a large-scale MNH initiative — a multi-country philanthropic program pooling resources from multiple global health foundations to accelerate maternal and newborn survival across sub-Saharan Africa.

The scale of the problem:

Metric Annual Deaths in Sub-Saharan Africa
Maternal deaths 182,000 (70% of global total)
Newborn deaths 950,000 (47% of global total)
Stillbirths 1.2 million (48% of global total)

Target: Save hundreds of thousands of lives over the program period across multiple countries.

The Causal Investment Question

Such an initiative typically invests in three pillars:

People
Health workforce
Products
MNH innovations
Systems
Data & referrals

The fundamental question is causal: Which combination of investments across these three pillars will cause the greatest reduction in mortality?

This is not a question that correlation analysis can answer.

Why Naive Data Analysis Fails

Suppose we compare spending vs. outcomes across the target countries:

Naive Correlation

Country A receives the most funding and has the highest absolute deaths.

“More spending = more deaths!”

This ignores that Country A is invested in because it has the highest burden.

Causal Reasoning

Country A has the highest burden and the greatest potential for lives saved.

The investment is targeted at the cause of mortality, not correlated with it.

  • Workforce, equipment, and data are mediators through which investment works
  • Quality of care is the proximate driver of outcomes
  • Confounders not shown: baseline disease burden, government capacity, political stability

Using dagitty and ggdag in R

These packages form our core toolkit throughout the course:

  • dagitty — specify and analyze DAGs (d-separation, adjustment sets)
  • ggdag — beautiful DAG visualizations using ggplot2
  • bnlearn — Bayesian network structure learning and inference (Ch 4+)

R: A Simple MNH Causal Model

# Define a simple 3-node DAG
mnh_simple <- dagitty('dag {
  Investment [pos="0,0"]
  Coverage [pos="1,0"]
  NMR [pos="2,0"]

  Investment -> Coverage
  Coverage -> NMR
}')

# Visualize
ggdag(mnh_simple) +
  theme_dag() +
  labs(title = "Simple MNH Causal Model",
       subtitle = "Investment → Coverage → Neonatal Mortality Rate")

R: A Confounded Model

# Hospital delivery and mortality: confounded by complication severity
confounded_dag <- dagitty('dag {
  Complication_Severity [pos="1,0"]
  Hospital_Delivery [pos="0,1"]
  Neonatal_Mortality [pos="2,1"]

  Complication_Severity -> Hospital_Delivery
  Complication_Severity -> Neonatal_Mortality
  Hospital_Delivery -> Neonatal_Mortality
}')

# Visualize with exposure and outcome
ggdag_status(confounded_dag,
             exposure = "Hospital_Delivery",
             outcome = "Neonatal_Mortality") +
  theme_dag() +
  scale_color_manual(values = c("exposure" = "#83BD00",
                                "outcome" = "#E24A3F",
                                "other" = "#F8A623")) +
  labs(title = "Confounding in Hospital Delivery Data")

R: Finding Adjustment Sets

# What must we control for to estimate the causal effect
# of hospital delivery on mortality?
adjustmentSets(confounded_dag,
               exposure = "Hospital_Delivery",
               outcome = "Neonatal_Mortality")
# Returns: { Complication_Severity }

Interpretation: To estimate the true causal effect of hospital delivery on neonatal mortality, we must control for complication severity. Without this adjustment, the correlation between hospital delivery and mortality is biased by confounding.

# MNH initiative theory of change
mnh_toc <- dagitty('dag {

  Workforce [pos="1,0"]
  Equipment [pos="1,1"]
  Data_Systems [pos="1,2"]

  Equipment -> Quality

# Identify the adjustment set for Budget → NMR

# Visualize
ggdag(mnh_toc) +
  theme_dag() +
  labs(title = "MNH Initiative: Theory of Change as a DAG")

Key Takeaways

Causal modeling serves two purposes: situational assessment (What is happening?) and managerial intervention (What should I do?).

Correlation ≠ causation. Confounding, reverse causation, and selection bias all create misleading associations.

A causal model makes your assumptions explicit, testable, and actionable — essential for allocating large-scale program funding effectively.
