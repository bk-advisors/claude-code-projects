From Intuition to Structure

Why writing down causal beliefs matters

The Problem with Mental Models

Every manager carries implicit causal beliefs about how the world works.

Implicit Models

  • Vague and inconsistent
  • Resistant to scrutiny
  • Hidden assumptions
  • Two people can disagree without realizing why

Explicit Models (DAGs)

  • Precise and shareable
  • Open to critique and testing
  • Assumptions are visible
  • Disagreements become productive

Core Principle: A qualitative causal model is a disciplined way of writing down what you believe about how variables in a system relate to one another — before you add any numbers.

Nodes

The four building blocks of causal diagrams

Node Type 1: Probabilistic Nodes (Ovals)

Probabilistic Node (Oval): A variable whose state is uncertain and determined by nature or context — not by any agent’s deliberate choice. Represented as an oval (or circle) in the diagram.

MNH Examples

  • Neonatal Mortality Rate (NMR): States = {High (>25/1000), Low (<=25/1000)}
  • Baseline Disease Burden: States = {Severe, Moderate, Low}
  • Workforce Quality: States = {Competent, Developing, Inadequate}

Key rule: Each probabilistic node must have MECE states — Mutually Exclusive and Collectively Exhaustive. The variable must be in exactly one state at any given time.

Node Type 2: Objective Nodes (Hexagons)

Objective Node (Hexagon): A variable representing a goal or performance metric that a decision-maker is trying to optimize. Represented as a hexagon. Objective nodes are always leaf nodes — nothing flows out of them.

MNH Examples

  • Lives Saved: The Meridian Health Alliance’s ultimate objective — save 322,000+ lives by 2030
  • Maternal Mortality Ratio (MMR): Measured in deaths per 100,000 live births
  • Cost-Effectiveness: Lives saved per dollar invested

Objective nodes sit at the end of causal chains. They receive arrows but never send them. They answer: What are we ultimately trying to affect?

Node Type 3: Strategic Option Nodes (Rectangles)

Strategic Option Node (Rectangle): A variable representing a deliberate choice made by a decision-maker. The states are the available options. Represented as a rectangle.

MNH Examples

  • Budget Allocation: States = {People-heavy, Products-heavy, Systems-heavy, Balanced}
  • Training Investment: States = {Fund, Don’t Fund}
  • Equipment Procurement: States = {CPAP, PPH Bundle, Ultrasound, Combination}

Strategic option nodes are the levers that the Meridian Health Alliance controls. These are root nodes — no causal arrows point into them (except informational links).

Node Type 4: Function Nodes (Chevrons)

Function Node (Chevron): A variable whose state is deterministically determined by its parents — there is no residual uncertainty. Represented as a chevron (or pointed rectangle). If you know all parent states, you know this node’s state with certainty.

MNH Examples

  • Total Budget Spent = Workforce spending + Equipment spending + Systems spending (a simple sum)
  • Coverage Rate = Facilities equipped / Total facilities (a deterministic ratio)

Probabilistic Node

Parent states → probability distribution over child states (uncertainty remains)

Function Node

Parent states → exactly one child state (no uncertainty)

Function nodes are rare in qualitative models but become important in quantitative models (Ch 4) where they represent accounting identities, formulas, or physical laws.

Directed Links

Arrows, signs, and the language of causation

Causal Links with Signs

Directed Link (Arrow): An arrow from node A to node B means that A is a direct cause of B — changing A can change B (holding all other parents of B fixed). The sign [+] or [-] indicates the direction of effect.

Sign Meaning Example
[+] Increasing A increases B (or being in “good” state of A makes “good” state of B more likely) Training [+] → Quality
[-] Increasing A decreases B (or being in “good” state of A makes “bad” state of B less likely) Quality [-] → Mortality
[0] A has no effect on B in this combination (used in conditional certainty tables) —

  • Root nodes have no parents — they are exogenous (Budget, Baseline Burden)
  • Leaf nodes have no children — they are terminal outcomes (NMR, MMR)
  • The directed path Budget → Training → Quality → NMR has an overall sign of [+] x [+] x [-] = [-] (more budget leads to lower mortality)

Sign Multiplication Along Paths

Sign Rule: The overall sign of a directed path is the product of individual link signs, where [+] x [+] = [+], [-] x [-] = [+], and [+] x [-] = [-].

Path Link Signs Overall Sign Interpretation
Budget → Training → Quality → NMR [+] x [+] x [-] [-] More budget reduces NMR
Budget → Equipment → Quality → NMR [+] x [+] x [-] [-] Same direction
Disease Burden → Coverage → NMR [-] x [-] [+] Higher burden → higher NMR

When multiple paths connect two nodes, the overall effect is ambiguous if paths have different signs. This is one reason we need quantitative models (Ch 4) — qualitative signs alone cannot resolve conflicting path effects.

Conditional Certainty Tables

When [+] and [-] are not enough

When Relationships Are Not Monotonic

A single [+] or [-] sign assumes the relationship is monotonic — more of A always means more (or less) of B. But some causal relationships are more complex.

Conditional Certainty Table: A table showing the qualitative sign ([+], [-], or [0]) of a child variable for each combination of parent states. Used when the effect of one parent depends on the state of another parent.

MNH Example: Effect of CPAP Machines on NMR

The effect of CPAP depends on whether trained staff are available to operate them:

Staff Trained Staff Not Trained
CPAP Available [-] (reduces NMR) [0] (no effect — machines sit idle)
CPAP Unavailable [0] (training alone insufficient) [0] (no intervention)

Reading Conditional Certainty Tables

How to read: Each cell answers the question: “If this parent combination holds, what is the qualitative direction of effect on the child?”

A richer example — effect on Quality of Care:

Equipment Available Equipment Unavailable
Staff Competent [+] strong positive [+] moderate (skill compensates)
Staff Developing [+] moderate (equipment helps) [0] weak or no effect
Staff Inadequate [0] equipment cannot compensate [-] active harm risk

Key Insight: Conditional certainty tables reveal interaction effects — situations where one input only works when combined with another. For the Meridian Health Alliance, this means People + Products investments are complements, not substitutes.

Serial Triplets (Chain)

A → B → C — information flows along the chain

The Serial Triplet Structure

Serial Triplet (Chain): Three nodes connected as A → B → C. Node B is a mediator — it transmits the causal effect of A to C.

A
→
B
→
C

Information flow rules:

  • Unconditionally: Information flows from A to C through B. Learning about A tells you something about C.
  • Conditioning on B blocks the flow. If you know B’s state, then learning A tells you nothing new about C.

Intuition: If I know the mediator’s exact value, the upstream cause becomes irrelevant for predicting the downstream effect.

MNH Example: Training → Competency → Mortality

Not conditioning on Competency:

Training and Mortality are associated. We can predict lower mortality from knowing training was completed.

Conditioning on Competency:

If we know a midwife is competent, it no longer matters how she became competent (training vs. experience). The path is blocked.

For the Meridian Health Alliance: This tells us that training’s value is entirely mediated through competency. If we can measure competency directly, we do not need training data to predict mortality.

Chain Summary

Condition A and C associated? Why?
Not conditioning on B Yes Information flows A → B → C
Conditioning on B No Knowing B blocks the path

Think of it as a pipeline

Water flows from the reservoir (A) through the pipe (B) to the tap (C). If you close the valve at B (condition on it), nothing from A reaches C.

Diverging Triplets (Fork)

A ← B → C — the common cause structure

The Diverging Triplet Structure

Diverging Triplet (Fork): Three nodes connected as A ← B → C. Node B is a common cause of both A and C. This creates a spurious correlation between A and C.

A
←
B
→
C

Information flow rules:

  • Unconditionally: A and C appear correlated — but only because B drives both. This is confounding.
  • Conditioning on B blocks the flow. Once you know the common cause, A and C become independent.

Intuition: Ice cream sales (A) and drowning deaths (C) are correlated — but the common cause is summer heat (B). Knowing the temperature eliminates the spurious link.

Naive observation:

Facilities with high PPH detection also have high CPAP usage. Does detecting PPH cause better CPAP use?

Causal reality:

Both are driven by workforce quality. Competent staff detect PPH and use CPAP correctly. Controlling for workforce quality eliminates the association.

For the Meridian Health Alliance: If we see a correlation between two program metrics, we must ask — is there a common driver? Investing in PPH protocols will not improve CPAP usage unless we address the underlying workforce quality.

Fork Summary

Condition A and C associated? Why?
Not conditioning on B Yes Common cause creates spurious correlation
Conditioning on B No Controlling for common cause removes confounding

This is exactly the confounding trap from Chapter 1! The fork structure is the formal representation of confounding. The solution is always: condition on the common cause.

Converging Triplets (Collider)

A → B ← C — the dangerous exception

The Converging Triplet Structure

Converging Triplet (Collider): Three nodes connected as A → B ← C. Node B is a collider — two causal arrows “collide” at B. This structure behaves opposite to chains and forks.

A
→
B
←
C

Information flow rules (REVERSED from chain and fork!):

  • Unconditionally: A and C are independent. No information flows — the path is naturally blocked at the collider.
  • Conditioning on B (or any descendant of B) OPENS the path. A and C become associated — this is collider bias.

Warning: Conditioning on a collider creates a spurious association that did not exist before. This is the opposite of what happens with chains and forks!

The “Explaining Away” Effect

Among facilities with good quality of care (conditioning on the collider), staffing and equipment become negatively associated: if quality is good despite low staffing, we infer equipment must be excellent — and vice versa. Each cause “explains away” the other.

Not conditioning on Quality:

Staffing and Equipment are independent (separate investment decisions).

Conditioning on Quality = Good:

Among good-quality facilities: low staffing → must have good equipment. Creates a spurious negative association.

Collider Summary and the “Explain Away” Intuition

Condition A and C associated? Why?
Not conditioning on B No Path is blocked at the collider
Conditioning on B Yes Collider bias — “explaining away”

The “explain away” intuition: Imagine you know a student passed an exam (collider). If you then learn the exam was easy, you revise downward your belief about the student’s talent. The two causes compete to explain the common effect.

c-Independence and d-Separation

Formal criteria for when information flows

From Triplets to Full Graphs

In a simple triplet, the rules are straightforward. But real DAGs have many paths between any two nodes, and each path may contain multiple triplet structures.

d-Separation (Directional Separation): Two nodes X and Y are d-separated given a set of conditioning variables Z if every path between X and Y is blocked. If d-separated, X and Y are conditionally independent given Z.

A path between X and Y is blocked by conditioning set Z if the path contains:

  • A chain (A → B → C) or fork (A ← B → C) where the middle node B is in Z, OR
  • A collider (A → B ← C) where neither B nor any descendant of B is in Z

Applying d-Separation: An Example

Consider this DAG:

Budget
→
Training
→
Quality
→
NMR
↑
Equipment

Q: Is Training d-separated from NMR given Quality?

Path: Training → Quality → NMR

Quality is in the conditioning set and is a chain middle node → Blocked.

Answer: Yes, d-separated.

Q: Is Training d-separated from Equipment given Quality?

Path: Training → Quality ← Equipment

Quality is a collider and is in the conditioning set → Opened.

Answer: No, not d-separated.

d-Separation Decision Procedure

1
List all paths
between X and Y
→
2
For each path, identify
every triplet structure
→
3
Check if any triplet
blocks the path
→
4
If ALL paths blocked:
d-separated

A path is blocked if it contains at least one:

  • Chain or fork with middle node in the conditioning set, OR
  • Collider with middle node (and all its descendants) not in the conditioning set

d-Separation holds only if EVERY path is blocked.

Why this matters: d-Separation tells us which conditional independencies our causal model implies. These are testable predictions — if the data violates them, our model is wrong.

Identifying Triplets Within the Model

Serial (Chain) Examples

  • Budget → Training → Quality
  • Referral System → Coverage → Quality
  • Budget → Workforce → Quality → NMR

Conditioning on the mediator blocks the path.

Diverging (Fork) Examples

  • Workforce ← Budget → Equipment
  • NMR ← Quality → MMR
  • Training ← Budget → Referral System

Budget is a common cause. Conditioning on it blocks spurious correlations between its children.

Converging (Collider) Examples

  • Workforce → Quality ← Equipment
  • Training → Quality ← Equipment (via Training → Quality)
  • Coverage → Quality ← Workforce

Quality is a collider. Conditioning on Quality creates a spurious association between Workforce and Equipment (“explaining away”).

Testable Implications

Our qualitative DAG implies specific conditional independencies that we can test against data:

  • Training ⊥ Equipment | Budget — given budget allocation, training and equipment decisions are independent (fork structure)
  • Workforce ⊥ Referral System | Budget — same logic; both are children of budget
  • Training ⊥ NMR | Quality — given quality of care, training has no additional direct effect on NMR (chain structure)
  • Workforce ⊥ Equipment | Quality — conditioning on the collider violates this; they should be dependent!

If the data violates any implied independence, our model is wrong and must be revised.

This is how qualitative models generate testable predictions — even without numbers.

R: Test d-Separation

# Test 1: Is Training d-separated from NMR given Quality?
dseparated(mnh_dag, "Training", "NMR", "Quality")
# Expected: TRUE (chain is blocked)

# Test 2: Is Workforce d-separated from Equipment? (unconditional)
dseparated(mnh_dag, "Workforce", "Equipment", c())
# Expected: FALSE (common parent Budget creates fork)

# Test 3: Is Workforce d-separated from Equipment given Budget?
dseparated(mnh_dag, "Workforce", "Equipment", "Budget")
# Expected: TRUE (conditioning on common cause blocks fork)

# Test 4: Is Workforce d-separated from Equipment given Quality?
dseparated(mnh_dag, "Workforce", "Equipment", "Quality")
# Expected: FALSE (conditioning on collider opens path!)

# Test 5: Is Training d-separated from Baseline_Burden?
dseparated(mnh_dag, "Training", "Baseline_Burden", c())
# Expected: TRUE (no connecting path)

Interpret each result: TRUE means conditionally independent; FALSE means information flows between the two variables given the conditioning set.

Key Takeaways

Qualitative causal models are the foundation of rigorous analysis.

A DAG encodes your causal beliefs explicitly using four node types and signed directed links.

The three triplet structures — chain, fork, and collider — govern all information flow. Chains and forks are blocked by conditioning; colliders are opened by conditioning.

d-Separation translates graph structure into testable conditional independence predictions.

For the Meridian Health Alliance: the qualitative DAG reveals that People, Products, and Systems investments are complements flowing through Quality of Care — and that conditioning on Quality creates collider bias between investment pillars.
